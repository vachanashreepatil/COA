~Instruction Format~

1. Immediate Addressing Mode

The operand is directly specified in the instruction itself.

MOV AX, 1234H
ADD AL, 56H

Advantage: No memory access required for operand fetch.

2. Direct Addressing Mode

The instruction contains the actual address of the operand.

MOV AX, [1234H]
ADD AL, [5678H]

Used when we know the exact memory location of data.

3. Register Addressing Mode

The operand is stored in a CPU register.

MOV AX, BX
ADD CX, DX

Fastest addressing mode as no memory access is required.

4. Register Indirect Addressing Mode

Register contains the address where the operand is stored.

MOV AX, [BX]
ADD CX, [SI]

Useful for accessing array elements and pointer operations.

5. Indexed Addressing Mode

The effective address is sum of base address and index register.

MOV AX, [BASE + SI]
ADD BX, [1000H + DI]

Commonly used in array operations.

6. Based Addressing Mode

Similar to indexed, but uses a base register instead.

MOV AX, [BX + 1234H]
ADD CX, [BP + 5678H]

Useful in accessing structure members and stack operations.

7. Based Indexed Addressing Mode

Combines base register, index register, and displacement.

MOV AX, [BX + SI + 1234H]
ADD CX, [BP + DI + 5678H]

Most flexible addressing mode, good for complex data structures.

8. Relative Addressing Mode

The offset is added to the Program Counter (PC).

JMP LABEL
BNE +12

Commonly used in branch and jump instructions.
